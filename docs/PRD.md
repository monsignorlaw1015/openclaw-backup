# OpenClaw 备份助手 PRD

## 产品定位

一个面向本地 OpenClaw 用户的桌面备份与迁移工具，目标是让普通用户也能完成换电脑迁移，而不需要手动翻目录、记路径、重配全部环境。

## 本期范围

- 平台：macOS
- 形态：本地桌面应用
- 范围：P0 可用版本
- 核心目标：打通真实目录结构下的“扫描 -> 导出 -> 预检查 -> 恢复”链路

## 真实痛点

- 用户不知道 `~/.openclaw` 里哪些内容必须备份
- 手工迁移时容易漏掉认证配置、记忆、会话历史和扩展
- 新旧电脑版本不一致时，恢复后可能出现配置兼容问题
- 敏感信息要么全部拷走有风险，要么不拷又要重配
- 导入前没有一眼可懂的风险提示

## 当前实现能力

### 1. 扫描真实可迁移项目

当前实现不是停留在“固定 5 项”的概念模型，而是对齐真实目录结构，识别这些内容：

- `openclaw.json`
- `agents/main/agent/auth-profiles.json`
- `identity/`
- `memory/`
- `skills/`
- `workspace/`
- `agents/main/agent/models.json`
- `agents/main/sessions/`
- `extensions/`
- `cron/jobs.json`

### 2. 导出备份包

- 备份格式：`.zip`
- 包含 `manifest.json`
- 仅导出用户勾选且有效的项目
- 支持选择是否包含敏感信息

### 3. 敏感信息策略

- 如果选择“包含敏感信息”，认证配置、设备身份等会原样导出
- 如果选择“排除敏感信息”，纯敏感项会被排除
- `openclaw.json` 会保留，但里面的密钥字段会被脱敏

### 4. 导入预检查

- 读取备份包内容
- 提示 manifest 缺失
- 提示版本差异
- 检测 OpenClaw 可能正在运行
- 提示备份包是否包含敏感信息
- 提示哪些项目已被脱敏

### 5. 恢复执行

- 覆盖式恢复
- 单项失败继续
- 恢复后做文件级校验
- 给出成功、告警、失败结果
- 提示用户手动重启 OpenClaw

## 非目标

- Windows 正式支持
- 自动重启 OpenClaw
- 云端 OpenClaw 迁移
- 增量备份
- Apple 签名与 notarization

## 验收标准

- 能识别真实 `~/.openclaw` 目录中的关键可迁移内容
- 能生成有效 zip 备份包和 `manifest.json`
- 能在导入前完成预检查并给出风险提示
- 能在临时目录完成真实导出恢复链路
- 脱敏模式下，`openclaw.json` 中的敏感字段必须被替换
- 排除敏感信息模式下，纯敏感项不能被写入备份包
